## A trillion tokens of code nobody wrote
A: Second real story, and it's mine. IBM Research has released something called CodeAlchemy. Nearly one trillion tokens of code, across fifteen programming languages, under a permissive license — and they released the pipeline and the prompts that made it, too.

A: Their comparison, and it's a good one: at least two hundred times larger than Wikipedia, which they put at about five billion tokens.

A: Generated. Sourced from three public repositories, then elaborated and rewritten by models — they used the GPT-O-S-S models for that part. But the volume is not what's interesting about it. B: Meaning they ran the code and kept the recording.
A: They ran the code and kept the recording. And a trace, in their description, shows a program's changing states, data transformations, and interactions with the real world through library APIs.
B: Okay, so give me the why. Why does that matter enough to build a trillion tokens?
A: Because of a gap that I think is the most quietly important idea in this story, and IBM state it in one sentence. Quote: downloading billions of lines of code from GitHub teaches a model what code looks like — but it doesn't teach what happens behind the scenes when you execute that code on different inputs.
B: [pause] Say more, because I think that sentence is doing a lot of work.
A: It's doing all the work. Think about what a model trained on GitHub has actually seen. It has seen the text of programs. Millions of them. It has learned, extremely well, what a plausible next line of a program looks like — given this function name and these imports and this indentation, here's what a competent programmer writes next. That is a real skill and it is genuinely useful and it is not the same skill as knowing what the program does.
B: Because the thing it does is never in the file.
A: The thing it does is never in the file. The file is a description of a process. The process happens somewhere else, at a different time, on inputs the file has never seen. And nothing in the training data connects the description to the outcome. B: [wry] Which is a hell of a way to learn what a program does.
