B: And I'm Alan. Google threw its biggest keynote of the year this morning, so there's a fair amount of Google in the show today — no apologies, it's genuinely where the news was. ## An AI agent leaked a private repo
A: So this is a security write-up from a firm called Noma Security, and I want to walk through it slowly, because every step is mundane and the sum is alarming. Here's the setup. GitHub — where basically the world keeps its code — rolled out something called Agentic Workflows. You pair GitHub's automation with an AI agent, Claude or Copilot under the hood, and you give it instructions in plain English. Somebody files an issue, the agent reads it, calls some tools, and acts on its own.
B: Right, and that's a genuinely useful pattern. "Someone reports a bug, an agent triages it" — half the teams I know want exactly that.
A: They do. Now here's the attack. A researcher goes to a public repository in some organization and opens an issue — looks completely ordinary, a normal bug report. But buried in the body of that issue are hidden instructions. Not a bug report at all. Commands.
B: Commands aimed at the agent, not the human.
A: Exactly. And when the workflow fires — it triggers the moment an issue gets assigned — the agent reads that issue body and treats the whole thing as instructions it should follow. It cannot tell the difference between "here is a problem to look at" and "here is what you should now go do."
B: [skeptical] Okay, but surely GitHub saw this coming. There have to be guardrails on "don't just do whatever a random issue tells you."
A: There are. And here is the detail I can't stop thinking about. The researcher found that if you start the malicious instruction with the word "Additionally" — just that, one transition word — the guardrails let it through. Their own phrasing: by tricking the model that way, they ensured GitHub's guardrails "did not work as intended."
B: [pause] The safety check got beaten by the word "additionally."
A: By a word you'd use in a cover letter. And once it's through, the payload is almost boring. The instructions tell the agent to go fetch the README files from the repositories it can reach — and post them as public comments on that issue.
B: And "the repositories it can reach" is the whole ballgame, isn't it.
A: That's the ballgame. This particular agent had been configured with read access across the organization — including private repositories. So it dutifully walks over to a private repo, reads the file, and pastes the contents into a comment on a public issue. Which anyone on the internet can now read.
B: [flat] So the attacker never touched the private repo. The agent did it for them, and helpfully published the result.
A: That's the shape of it. And notice what the attacker actually needed. No password. No stolen key. No coding ability. The only requirement was the ability to open an issue on any public repo in that organization — which is to say, none. Noma disclosed it to GitHub responsibly, and published with their knowledge, so this is a fixed, known thing, not a live zero-day. But the pattern is the lesson.
B: And the pattern isn't new, is it. This is the thing you keep saying every injection story is secretly about.
A: It's the confused deputy, and it's old. Go back to the seventies — the security literature already had a name for a program that holds real authority and gets talked, by an untrusted party, into using that authority on their behalf. The deputy isn't malicious. It's confused about whose orders it's following. And prompt injection is that exact bug, wearing new clothes. It's the direct descendant of SQL injection from the database world — where the fix, drilled into every engineer for twenty years, was: never let data cross the line into becoming commands.
B: And an AI agent smashes that line on purpose.
A: That's the cruel part. The entire premise of these agents is that instructions and information arrive in the same channel — natural language — and the model figures out which is which. So we've taken the one architectural rule that made systems safe, and built a product whose core feature is breaking it. The issue body is data. The agent read it as a command. Every guardrail is just trying to reconstruct, after the fact, a boundary the design threw away on purpose.
B: So let me land the so-what for anyone actually running these, because it's very concrete. If you wire an agent to trigger on input from people you don't control — issues, emails, tickets, pull requests from strangers — and you hand that agent broad permissions, you have built a confused deputy. It doesn't matter how good the model is.
A: And "how good the model is" almost makes it worse. A more capable agent carries out the injected instructions more competently.
B: Right. The mitigation is the unglamorous one — least privilege. That agent did not need read access to every private repo in the org to triage a bug. The blast radius was a permissions decision, made once, probably by someone who wasn't picturing an attacker three steps downstream.
A: That's the note I'd leave it on. The exploit is clever, but the fix isn't a cleverer guardrail — it's assuming the agent will be fooled, and making sure that when it is, there's almost nothing valuable within its reach.
## The thing Google actually shipped
B: Alright, my turn, and this one's a nice antidote — something real, that shipped today, that you can put your hands on. In the middle of all the Google keynote noise, they released Gemma four. New generation of their open-weight models, Apache-two licensed — which is the part that matters, because Apache-two means you can genuinely build on it, commercially, without asking anyone.
A: And "open weight" here means the actual model, downloadable — not an API you rent.
B: The weights themselves. It comes in a spread of sizes — from a little two-billion-parameter one all the way up to a thirty-one billion dense model, plus a mixture-of-experts version that's twenty-six billion total but only lights up about four billion at a time. So there's a size for a laptop and a size for a serious box.
A: Give me the interesting part, though, because "Google released more Gemma" is not, by itself, a story.
B: [chuckles] Fair. The interesting part is one word in their technical report: encoder-free. So — normally, when you make a language model see and hear, you bolt on separate specialist parts. A vision encoder, trained on its own, whose whole job is to turn an image into something the language model can read. Another one for audio. The language model in the middle never actually touches a pixel — it gets a pre-digested summary from these bolted-on translators.
A: The lineage there goes back to the models that first fused vision and language — a separately-trained image encoder, stitched to a language model with a little adapter in between. That's been the standard recipe for years.
B: Right, and Gemma four's twelve-billion model throws the translators out. No separate encoders. It takes raw audio — in little forty-millisecond slices — and raw image patches, and feeds them nearly straight into the model through a thin, lightweight projection. One model, eating pixels and sound directly.
A: [interested] So instead of a committee of specialists handing notes to a generalist, it's one system learning to read all of it itself.
B: That's the shift. Fewer moving parts, one thing to serve and deploy, and the images and audio aren't second-class citizens filtered through a translator. And they paired it with the efficiency trick that actually makes it runnable — for every five layers doing cheap, local attention, only one does the expensive global kind that looks across the whole context. They report that ratio cuts the memory their long-context cache needs by more than a third.
A: Which is the difference between "supports long context on a slide" and "supports long context on your actual hardware."
B: Exactly that difference. There's also a thinking mode — it'll generate reasoning steps before answering when you want that. And the numbers, from their own report: the thirty-one billion model scores around eighty-five on a broad reasoning benchmark, high-eighties on a hard 2026 math set. They call it the leading dense open model.
A: [skeptical] Alan. "Leading dense open model." Count the qualifiers. Dense — so, not counting the mixture-of-experts models. Open — not counting the closed frontier ones. That's a sentence engineered to carve out a lane narrow enough to win.
B: [laughs] It absolutely is, and you're right to catch it. And the honest tell is in their own citation — on the big public head-to-head leaderboard, this model sits around forty-third overall. So there are a lot of models above it. The claim isn't "this beats the frontier." It's "among models of this specific, useful kind — the ones you can download, run, and modify — this is the strong one."
A: And that's a fair claim, stated honestly, which I'll take. Because the category it's winning is the one a lot of people actually live in — you can't fine-tune a model you're only allowed to reach through an API.
B: Which is the whole reason it's the story and the keynote demos aren't. Google spent the morning showing agents that are coming to trusted testers, coming next month, coming this summer. This is the one thing from the entire day a developer can download this afternoon and start changing.
A: The download beat the demo.
B: The download beat the demo. Now — thirty-one billion parameters is still a real graphics card. But the two and four-billion versions run on a decent laptop, and that end of the range is where "modify it yourself" stops being a slogan.
## The sweep

B: We do. So — Google I/O. Sundar Pichai's line was that they're "firmly in our agentic Gemini era," and the framing all day was agents that run around the clock in the background and do things for you. The scale numbers are genuinely staggering — they say the Gemini app is at nine hundred million monthly users, and they're processing north of three quadrillion tokens a month.
A: So what's real today versus what's a promise?
B: That's the honest split. Two things are shipping now. The coding-agent wars are spilling into the rest of the office.
